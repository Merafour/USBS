1.为了支持苹果系统，默认按照 48K RAM设计程序，如果你所使用的 IC RAM低于 48K，后续将提供按照 20K RAM设计的程序。  
-----2015.4.17
2. Travel版本为增加用户使用支持通过 Config.bin文件配置 IO，即将程序烧进去之后把 Config.bin文件下载进去重新上电即可配置 IO。
   Config.bin为 2.6版本配置软件新增的功能，选择 Travel进行配置即可得到该配置文件。
   生成 Config.bin无需点击 "Choose"选择文件。
   通过 Config.bin配置 IO将不能为 App提供的加密，不是技术上不能实现，而是考虑到安全性暂时没有去实现。
-----2015.4.20
3. 从 Travel版本 V1.24.6版本开始提供 “User Data”功能,适用于没有外部存储而 App需要保存数据的情况,配置软件请使用 2.8以上版本。
   从 V1.24.6版本开始通过 RAM扩大了我们在格式化的时候使用的容量，比如我是 256KB的 IC，但格式化的时候却是 274KB，也就是多出了 18KB，格式化完成之后容量为 254KB，
在这 254KB中，有 18KB是 RAM空间，也就是实际能够保存在 Flash中的数据为 236KB，也就是说 App不能超过 236KB。倘若你设置了 “User Data”如 4K，那么你的固件最大只能是 232KB，
因为另外 4KB的空间用来给 App存放数据。
-----2015.5.29
4. Travel V1.24.8版本修改了升级过程中的LED提示，常态依然为 LED闪烁，升级过程中为 LED常量，以区分。
-----2015.6.1
5. QiFreeUSBS_V1.24.8_Travel_NoRAM.hex 为不带 RAM扩展的，因后面测试发现 RAM扩展在大容量(512K)时格式化不成功，故将 RAM扩展去掉了，小容量是没问题的。
